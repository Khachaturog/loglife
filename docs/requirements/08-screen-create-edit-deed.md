# 08. Экран: создание и редактирование дела

## Назначение

Экран, на котором пользователь создаёт новое дело или редактирует существующее: задаёт эмодзи/символы, название, описание и настраивает блоки вопрос-ответ (тип, порядок, обязательность).

## Элементы экрана

Экран разделён на **вкладки «Дело»** и **«Аналитика»** (настройки отображения блоков аналитики на карточке дела после сохранения).

### Вкладка «Аналитика»

- **Активность** (сводка по числовому блоку) — общий переключатель группы; при включении: выбор числового блока (`number` / `scale` / `duration`), по которому считается сумма; `null` — первый числовой блок по порядку формы. Три переключателя видимости карточек «Сегодня», «За месяц», «Всего». Если пользователь выключает все три, главный переключатель группы автоматически переходит в «выкл»; при повторном включении группы, если все три были выключены, снова включаются все три карточки (дефолтный набор).
- **Стрики и записи** — общий переключатель группы; стрики (и при включённых стриках — счётчик «максимум стрика»), блок «всего записей», вложенно при включённом «всего» — счётчик «будни · выходные». Переключатель будни/выходные скрыт без «Всего», но его значение в состоянии не сбрасывается при временном выключении «Всего» (как «максимум стрика» при выключенном стрике). Если выключены и стрик, и «Всего», главный переключатель группы автоматически выключается; при повторном включении группы при полностью выключенных подпунктах восстанавливаются дефолты (стрик, максимум, «всего», будни/выходные).
- **Тепловая карта** — общий переключатель блока; при включении: источник значения по дню (число записей или сумма по выбранному числовому блоку), цвет карточки и градиента карты — `card_color` (селект пресетов Radix шаг 9 + пикер; пусто — акцент темы). Дополнительно: подписи дней недели слева, подписи месяцев над сеткой, нижний ряд «пик» и легенда «Меньше» … «Больше».

Конфигурация аналитики сохраняется в `deeds.analytics_config` (JSON, версия 1); цвет карточки дела — в `deeds.card_color`.

### Поля дела

- **Эмодзи/символы** — выбор через emoji-mart picker (кнопка с текущим эмодзи, по клику — Popover с пикером).
- **Название** — текстовое поле.
- **Описание** — многострочное поле (textarea), необязательное. Высота поля подстраивается под объём текста (без лишней пустой высоты), но не больше 500px; при большем тексте прокрутка только внутри поля. Переносы строк сохраняются при вводе.
- **Категория** — необязательный выбор из списка или своё значение.

### Блоки вопрос-ответ

- Список блоков в текущем порядке.
- Для каждого блока:
  - Текст вопроса (заголовок блока).
  - Тип блока: Число, Текст (абзац), Один из списка, Несколько из списка, Шкала, Да/Нет, Время.
  - Настройки типа (для списков — варианты и их порядок; для шкалы — число делений, подписи первого и последнего значения).
  - Чекбокс или переключатель «Обязательное значение».
  - Управление порядком: перемещение блока вверх/вниз или drag-and-drop.
  - **Удаление блока** — кнопка для удаления блока. Доступно при создании и при редактировании дела.
- Кнопка/действие **«+ Добавить блок вопрос-ответ»** — добавление нового блока (тип выбирается пользователем, по умолчанию можно подставлять «Число» для первого блока при создании дела).

### Сохранение

- Кнопка «Сохранить» (или «Создать») — сохранение дела и возврат к списку дел или к карточке дела.

#### Условия, при которых сохранение доступно

- **Название дела** — непустое (после обрезки пробелов).
- **Хотя бы один блок** в списке.
- **Блоки «Один из списка» и «Несколько из списка»** — у каждого варианта должен быть введён собственный текст подписи (не только подсказка «Вариант 1», «Вариант 2» в поле ввода); вариантов должно быть не меньше одного.

Для **остальных типов блоков** (число, текст, шкала, да/нет, время) отдельных обязательных полей в конфиге на экране нет: пустой текст вопроса при сохранении подставляется как заголовок «Блок»; настройки шкалы и прочие поля — по умолчанию или пустые, если так в форме.

## Режимы

- **Создание**: все поля пустые, по умолчанию добавлен блок типа «Число».
- **Редактирование**: поля заполнены данными существующего дела, блоки отображаются в сохранённом порядке.

## Touch-зоны (mobile-first)

Интерактивные элементы должны иметь touch-зону не менее 44–48px по высоте/ширине для удобного нажатия на мобильных устройствах.
