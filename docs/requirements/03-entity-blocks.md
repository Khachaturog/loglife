# 03. Типы блоков вопрос-ответ

Каждый блок в деле имеет тип, который определяет способ ввода и отображения ответа.

## Число

Человек вводит одно числовое значение (целое или дробное). Пример: «Приседаний: 10», «Вес: 72.5».

- Один инпут для числа; значение по умолчанию в редакторе дела подчиняется тем же правилам валидности, что и при сохранении (ненулевое конечное число).

## Текст (абзац)

Свободный текст: короткий или длинный, с переносами. Многострочное поле: по умолчанию по высоте как одна строка, при вводе растёт по содержимому до лимита, затем прокрутка внутри поля.

## Один из списка

Пользователь при настройке блока добавляет варианты ответа. При заполнении формы выбирается только один вариант.

- Варианты в списке можно упорядочивать (см. [05-rules-ordering.md](05-rules-ordering.md)).
- В конфиге блока (`blocks.config`) хранится **`singleSelectUi`**: `select` (выпадающий список) или `checkbox` (список чекбоксов, визуально как у «Несколько из списка»); ответ по-прежнему один id варианта. На экране просмотра записи ответ показывается строкой текста, без этого переключателя.

## Несколько из списка

Пользователь добавляет варианты ответа. При заполнении можно выбрать любое количество вариантов.

- Варианты в списке можно упорядочивать в рамках блока.

## Шкала

Пользователь задаёт количество делений на шкале (максимум 10 делений). В настройках дела можно задать подписи для делений (в т.ч. для всех промежуточных — через «Заполнить все подписи»), чтобы ориентироваться при ответе.

- Пример: шкала от 1 до 5 с подписями «Плохо» и «Отлично».
- При заполнении и редактировании записи: если в конфиге есть **хотя бы одна** непустая подпись, под сегментами показывается строка с подписью выбранного деления (или число, если для этой позиции подпись не задана); пока значение не выбрано — подсказка «Выберите значение». Если подписей нет — показывается только сегментированный ряд с цифрами, без дополнительной строки.
- В просмотре записи и при экспорте при наличии подписей значение может отображаться в виде «n — подпись».

## Да/Нет

Факт отметки без ввода значения. Подходит для вопросов вида «сделал ли я задачу», «выполнено ли».

- При ответе один чекбокс с подписью «Выполнено»: включён — да, выключен — нет.
- На экране просмотра записи — неактивный чекбокс и текст «Выполнено» или «Не выполнено».

## Время

Длительность в формате `ЧЧ:ММ:СС` (например время тренировки).

---

## Общее для всех блоков

- У каждого блока есть настройка **«Обязательное значение»** (да/нет). См. [06-rules-required.md](06-rules-required.md).
- Опционально: **значение по умолчанию** — колонка `blocks.default_value` (формат как ответ в записи); флаг **`blocks.default_value_enabled`** — подставлять ли его при открытии формы новой записи (при выкл. значение в БД сохраняется как черновик). Пользователь может подправить поле перед сохранением.
- Для типов **«Число»** и **«Один из списка»**: переключатель **подсказок из недавних записей** (чипы; колонка `blocks.recent_suggestions_enabled`, по умолчанию включено).
- Блоки можно перемещать в нужном порядке относительно друг друга. См. [05-rules-ordering.md](05-rules-ordering.md).
