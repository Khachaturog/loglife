# 03. Типы блоков вопрос-ответ

Каждый блок в деле имеет тип, который определяет способ ввода и отображения ответа.

## Число

Человек вводит одно числовое значение. Пример: «Приседаний: 10».

- Один инпут для числа.

## Текст (абзац)

Свободный текст: короткий или длинный, с переносами. Многострочное поле: по умолчанию по высоте как одна строка, при вводе растёт по содержимому до лимита, затем прокрутка внутри поля.

## Один из списка

Пользователь при настройке блока добавляет варианты ответа. При заполнении формы выбирается только один вариант.

- Варианты в списке можно упорядочивать (см. [05-rules-ordering.md](05-rules-ordering.md)).

## Несколько из списка

Пользователь добавляет варианты ответа. При заполнении можно выбрать любое количество вариантов.

- Варианты в списке можно упорядочивать в рамках блока.

## Шкала

Пользователь задаёт количество делений на шкале (максимум 10 делений). Указываются заголовки для первого и последнего значения, чтобы удобнее ориентироваться при ответе.

- Пример: шкала от 1 до 5 с подписями «Плохо» и «Отлично».

## Да/Нет

Факт отметки без ввода значения. Подходит для вопросов вида «сделал ли я задачу», «выполнено ли».

- При ответе один чекбокс с подписью «Выполнено»: включён — да, выключен — нет.
- На экране просмотра записи — неактивный чекбокс и текст «Выполнено» или «Не выполнено».

## Время

Длительность в формате `ЧЧ:ММ:СС` (например время тренировки).

---

## Общее для всех блоков

- У каждого блока есть настройка **«Обязательное значение»** (да/нет). См. [06-rules-required.md](06-rules-required.md).
- Блоки можно перемещать в нужном порядке относительно друг друга. См. [05-rules-ordering.md](05-rules-ordering.md).
