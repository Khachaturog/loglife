# 04. Сущность «Запись»

## Определение

Запись — один момент заполнения формы по конкретному делу. Одна запись соответствует одной отправке формы и хранит ответы по всем блокам этого дела.

## Состав записи

### Ответы по блокам

Для каждого блока дела в записи сохраняется ответ в соответствии с типом блока:

- **Число** — одно число.
- **Текст (абзац)** — текст (в т.ч. с переносами строк).
- **Один из списка** — один выбранный вариант.
- **Несколько из списка** — набор выбранных вариантов.
- **Шкала** — выбранное значение на шкале.
- **Да/Нет** — факт «выполнено» / «не выполнено» (в интерфейсе — чекбокс; при просмотре — неактивный чекбокс и подпись).

### Связь с делом

Каждая запись привязана ровно к одному делу. По делу можно получить список всех записей (история). При редактировании дела (переименование, добавление/удаление блоков, смена типа блока) поведение данных в старых записях описано в [13-rules-editing-deed-data.md](13-rules-editing-deed-data.md).

На экране просмотра записи (`/records/:id`) в шапке есть действие **добавить новую запись** по тому же делу (переход на заполнение формы `/deeds/:deedId/fill`), чтобы после просмотра похожей записи можно было сразу создать ещё одну.

### Превью в списках (история, список записей по делу)

Если у дела **все** активные блоки имеют тип «Да/Нет», в превью строки записи показывается **«N из M»** — сколько блоков отмечено как выполненные (`yesNo: true`) из общего числа таких блоков. Иначе превью — краткие фрагменты ответов по блокам, склеенные через « · ».

### Служебные поля

У каждой записи есть технические поля, общие для всех дел:

- Дата добавления записи.
- Время добавления записи.
- Блок «Заметки» — технический блок внизу формы (свободный текст на всякий случай).

Подробнее см. [11-technical-defaults.md](11-technical-defaults.md).
