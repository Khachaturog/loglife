# 14. Экран: просмотр дела

## Назначение

Страница просмотра одного дела: пользователь видит информацию о деле и историю записей по нему, а также может выполнить действия (добавить запись, редактировать дело, удалить дело).

## Содержимое страницы

- **Эмодзи/символ** — визуальный идентификатор дела.
- **Название** — название дела.
- **Описание** — описание дела (если задано): переносы строк отображаются как в тексте; вставленные URL `http`/`https` показываются компактно (иконка и подпись «Ссылка») и открываются в новой вкладке; без отдельного сервиса сокращения ссылок.
- **История записей** — список записей по данному делу в хронологии (дата, время, превью ответов). Записи можно открывать для просмотра полных ответов и редактирования служебных полей (дата, время, заметки). Контекстное меню по карточке (long tap / ПКМ) и остальные сценарии — в [10-screen-history.md](10-screen-history.md).
- **Виджеты «Сегодня / За месяц / Всего»** (если включены в настройках аналитики дела): при выбранном **числовом** блоке в сводке — сумма значений по этому блоку; если числового блока в деле **нет** (например, только «Да/Нет»), те же три числа считаются как **количество записей** (сегодня, в текущем календарном месяце, всего), согласованно с превью на списке дел ([07-screen-list-deeds.md](07-screen-list-deeds.md)).
- **Теплокарта активности** (если включена): для **будущих** календарных дней ячейки не рисуются квадратами (остаётся выравнивание сетки по неделям; текущий день и прошлые — как раньше).

## Доступные действия

- **Добавить запись** — по умолчанию переход на экран заполнения формы ([09-screen-fill-form.md](09-screen-fill-form.md)). Если в [редакторе дела](08-screen-create-edit-deed.md) включено **быстрое добавление** и у каждого блока задан валидный дефолт: **короткое** нажатие «+» в шапке создаёт запись с этими значениями (дата/время — текущие локальные); **удержание** «+» открывает полную форму. Значения берутся из дефолтов блоков (в т.ч. «Нет» для «Да/Нет»). При ошибке быстрого сохранения — сообщение на странице. Прямой переход на URL формы по-прежнему возможен.
- **Редактировать дело** — переход на экран редактирования дела ([08-screen-create-edit-deed.md](08-screen-create-edit-deed.md)): изменение эмодзи, названия, описания, блоков вопрос-ответ.
- **Удалить дело** — удаление дела (с подтверждением). Поведение при удалении: удаляется запись.

## Навигация

- **Вход:** со списка дел ([07-screen-list-deeds.md](07-screen-list-deeds.md)) — по клику на карточку дела или по действию «Открыть» переходим на страницу просмотра дела.
- **Выход:** навигация назад — возврат к списку дел; с страницы просмотра дела переходы на экраны заполнения формы и редактирования дела, после сохранения можно возвращаться обратно на страницу просмотра дела или в список дел.
